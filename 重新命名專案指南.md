# 📝 專案重新命名指南

## 🎯 推薦的專案名稱

- `PinkshopOrderingSystem` - 品渴茶鋪訂購系統
- `PinkTeaShop` - 粉紅茶鋪
- `BeverageOrderingSystem` - 飲料訂購系統
- `OnlineTeaShop` - 線上茶鋪

---

## ⚠️ 注意事項

**直接更改資料夾名稱會導致：**
- ❌ Visual Studio無法開啟專案
- ❌ 命名空間不一致
- ❌ 編譯錯誤
- ❌ 路徑錯誤

**必須按照以下步驟進行！**

---

## 🔧 完整更改步驟

### 步驟 1：備份專案（重要！）

複製整個 `WebApplication1` 資料夾作為備份：
```
WebApplication1 → WebApplication1_backup
```

---

### 步驟 2：關閉Visual Studio

確保專案完全關閉，不要開啟任何檔案。

---

### 步驟 3：更改外層資料夾

```
C:\Users\User\Desktop\WebApplication1\
↓
C:\Users\User\Desktop\PinkshopOrderingSystem\
```

---

### 步驟 4：更改方案檔案名稱

在新的資料夾中：
```
WebApplication1.sln
↓
PinkshopOrderingSystem.sln
```

---

### 步驟 5：更改專案資料夾名稱

```
PinkshopOrderingSystem\WebApplication1\
↓
PinkshopOrderingSystem\PinkshopOrderingSystem\
```

---

### 步驟 6：更改專案檔案名稱

在專案資料夾中：
```
WebApplication1.csproj
↓
PinkshopOrderingSystem.csproj
```

---

### 步驟 7：編輯方案檔案 (.sln)

用記事本開啟 `PinkshopOrderingSystem.sln`

**找到這一行：**
```
Project("{9A19103F-16F7-4668-BE54-9A1E7A4F7556}") = "WebApplication1", "WebApplication1\WebApplication1.csproj", "{GUID}"
```

**改成：**
```
Project("{9A19103F-16F7-4668-BE54-9A1E7A4F7556}") = "PinkshopOrderingSystem", "PinkshopOrderingSystem\PinkshopOrderingSystem.csproj", "{GUID}"
```

**注意：保留原本的GUID不要改！**

儲存檔案。

---

### 步驟 8：開啟Visual Studio

雙擊新的 `PinkshopOrderingSystem.sln` 開啟專案。

如果出現錯誤，檢查步驟6和步驟7是否正確。

---

### 步驟 9：更改命名空間（重要！）

Visual Studio提供批次更改功能：

#### 方法A：使用搜尋取代（推薦）

1. 按 `Ctrl + Shift + H`（在檔案中取代）
2. 搜尋：`WebApplication1`
3. 取代為：`PinkshopOrderingSystem`
4. 範圍：整個方案
5. 點擊「全部取代」

#### 方法B：手動更改

需要更改的檔案：

**1. Controllers/HomeController.cs**
```csharp
namespace WebApplication1.Controllers
↓
namespace PinkshopOrderingSystem.Controllers
```

**2. Models/ErrorViewModel.cs**
```csharp
namespace WebApplication1.Models
↓
namespace PinkshopOrderingSystem.Models
```

**3. Program.cs**
（如果有using WebApplication1）
```csharp
using WebApplication1;
↓
using PinkshopOrderingSystem;
```

**4. Views/_ViewImports.cshtml**
```razor
@using WebApplication1
@using WebApplication1.Models
↓
@using PinkshopOrderingSystem
@using PinkshopOrderingSystem.Models
```

---

### 步驟 10：清理並重建

1. 在Visual Studio中，點擊 **「建置」→「清理方案」**
2. 刪除 `bin` 和 `obj` 資料夾
3. 點擊 **「建置」→「重建方案」**
4. 確認沒有錯誤

---

### 步驟 11：測試執行

1. 按 `F5` 執行專案
2. 測試所有功能
3. 確認一切正常

---

## 🎯 完整檢查清單

更改後請確認：

- [ ] 資料夾名稱已更改
- [ ] .sln 檔案名稱已更改
- [ ] .sln 檔案內容已更新
- [ ] 專案資料夾名稱已更改
- [ ] .csproj 檔案名稱已更改
- [ ] 所有 .cs 檔案的命名空間已更改
- [ ] _ViewImports.cshtml 已更改
- [ ] 專案可以成功編譯
- [ ] 專案可以正常執行
- [ ] 所有功能正常運作

---

## 🐛 常見問題

### Q1: 開啟方案時出現「找不到專案」

**原因：** .sln 檔案中的專案路徑不正確

**解決：** 用記事本編輯 .sln，確認專案路徑正確

### Q2: 編譯錯誤「找不到命名空間」

**原因：** 命名空間沒有完全更改

**解決：** 使用 `Ctrl + Shift + H` 全局搜尋 `WebApplication1` 並取代

### Q3: Views 無法找到 Controller

**原因：** _ViewImports.cshtml 沒有更新

**解決：** 檢查並更新 Views/_ViewImports.cshtml

## 💡 替代方案：重新建立專案（不推薦）

如果上述步驟太複雜，可以：

1. 建立新專案，使用正確的名稱
2. 將所有程式碼複製過去
3. 重新配置資料庫連線等設定

但這樣會很費時，**不建議**。

## 📋 建議的專案名稱格式

### 好的命名：
- ✅ `PinkshopOrderingSystem`
- ✅ `OnlineTeaShop`
- ✅ `BeverageShop`
- ✅ `TeaOrderingPlatform`

### 不好的命名：
- ❌ `test` - 太籠統
- ❌ `MyProject` - 不具體
- ❌ `專案1` - 避免中文
- ❌ `Pinkshop Order System` - 避免空格
